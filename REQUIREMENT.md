# 观鸟

**VIBE CODING**。

## 网站要有的功能

1. 用户能够上传观鸟图片，上传时将附带：
    - 鸟类名字（中文名、英文名和拼音，后两个可选）
    - 描述（可选）
    - 地点（从地图选择器上选择一个地点）
    - 拍摄时间
    - 拍摄人（默认为当前用户）
2. 用户进入网站后，将看到一张**中国地图**，用颜色标明每个省份的照片数量，鼠标触摸省份将显示这个省份的统计数据：
    - 照片数量
    - 鸟类数量
    - Top3鸟类
3. 用户可以依次点击进入省份、市区，在进入到省份级别及以下时，网页变成左右两栏，左边同上显示省份内信息，右边显示照片缩略图预览，触摸照片缩略图将显示照片简要信息（鸟类名称、拍摄人和拍摄时间）并在左边省份中标出相应地点
4. 用户点击照片后将弹出显示这张照片，以及照片的详细信息（鸟类名称、描述、地点、拍摄时间、拍摄人）
5. 有一个网页Gallery，显示当前上传的所有照片，可以进行筛选（省份、拍摄人）、搜索（鸟类名字、地点）
6. 用户可以登录，只有登录的用户可以上传照片，登录的用户可以管理自己的照片
7. 管理员用户可以通过后台管理所有照片，只有管理员可以注册用户
8. 【可选】加入自动识别照片中的鸟类（第三方API）

## 技术选型
1. 数据库选用 MySQL 数据库，通过 Docker 配置自动初始化
2. 前后端分离架构
    - 前端使用 Vue 3
    - 后端使用 Node.js + Express 或 python flask
3. Docker 容器化部署
    - docker-compose 一键启动多个服务
    - MySQL、后端、前端独立容器
    - 完整的健康检查和自动重启
    - 对外仅暴露80端口，docker内部转发，数据库在3306端口，后端在5000端口，前端在80端口
